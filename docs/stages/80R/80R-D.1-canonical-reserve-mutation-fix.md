# 80R-D.1 — Canonical Reserve Mutation Fix

**Stage:** 80R-D.1  
**Type:** Runtime fix (canonical state alignment)  
**Parent audit:** 80R-D canonical reserve mutation audit (read-only; conversation deliverable)  
**Depends on:** [80R-C runtime gate](./80R-C-runtime-gate.md), [80R-C.1 semantic continuity](./80R-C.1-savings-release-semantic-continuity.md)

---

## Objective

Stabilise Savings Release **canonical reserve truth** so `saved` / `balance` display fields stay aligned with `baseSaved` / `baseBalance` after confirm **and** after app reload.

---

## 80R-D root cause

| Finding | Detail |
|---------|--------|
| Apply is real | `geodeApplySavingsRelease` mutates `baseSaved` / `baseBalance`, recomputes, saves, appends `savingsReleases[]` |
| Not preview-only | 80R-C/C.1 changed copy/classification only; apply unchanged |
| Load gap | `load()` seeded base fields but **did not** call `geodeRecomputeBalancesFromPayments()` — display could show stale `saved` / `balance` |
| History hidden | `savingsReleases[]` persisted but no UI surface (deferred **80R-D.2**) |
| Plan UX gap | Overbudget block does not show reserve/net-worth delta; Monthly Left unchanged by design |

---

## Changes in 80R-D.1

### 1. Recompute on load

In `load()`, after `geodeNormalizeGoalInvestBaseFields` and `geodeNormalizeSavingsReleases`:

```javascript
geodeRecomputeBalancesFromPayments();
```

**Rationale:** On every boot, `saved` and `balance` = base fields + paid payment contributions. Fixes desync without new schema or migration.

### 2. Post-release balance confirmation toast

New helper: `geodeSavingsReleaseSuccessToastLine(preview, releaseEvent)`.

Example: `Recorded. Emergency buffer now shows £50.`

- Uses source name when available  
- Does **not** claim Monthly Left, overbudget, or plan changed  

### 3. Release record metadata

`geodeApplySavingsRelease` appends optional fields on new records:

- `reserveClass`
- `reserveTier`
- `copyKey`

`geodeNormalizeSavingsReleases` preserves these when present. Old records without them remain valid.

### 4. Dev audit hook (non-blocking)

After apply, sets `window._geodeLastSavingsReleaseAudit` and `console.warn` on remaining mismatch. Does not throw or block confirm.

## Fields touched

| Location | Change |

## Manual QA checklist

1. Open local app; note buffer saved (e.g. £60).
2. Release £10 from T0 buffer via Plan overbudget CTA.
3. Success toast includes remaining balance (e.g. £50).
4. Money → Goals shows £50 on buffer goal.
5. Home / Money Overview net worth down ~£10 (whole-pound rounding ok).
6. Hard refresh; buffer still £50.
7. DevTools: `S.savingsReleases.slice(-1)` has amount, sourceId, reserveClass/reserveTier.
8. `baseSaved` and `saved` aligned after reload.
9. Monthly Left unchanged.
10. Overbudget amount unchanged.
11. Pension/property (T4) still absent from release modal.
