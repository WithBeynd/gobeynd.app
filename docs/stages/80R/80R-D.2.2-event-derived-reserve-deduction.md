# 80R-D.2.2 — Event-Derived Reserve Deduction

**Stage:** 80R-D.2.2  
**Type:** Runtime fix (canonical reserve ledger)  
**Parent spec:** [80R-D.2.1 buffer canonical baseline alignment](./80R-D.2.1-canonical-reserve-mutation-fix.md) (conversation)  
**Depends on:** 80R-D.1 (recompute-on-load), 80R-C/C.1 (classification + copy)

---

## Objective

Fix Savings Release so confirmed releases reduce **effective** buffer / goal / investment balances via:

```text
effective reserve =
  baseSaved / baseBalance
  + Σ confirmed paid linked contributions
  − Σ confirmed event-derived savingsReleases for sourceId
```

Monthly Left, affordability, overbudget, and Plan sequencing remain unchanged.

---

## Root cause (80R-D.2.1)

- Apply subtracted `baseSaved` / `baseBalance` only.
- Recompute rebuilt `saved` / `balance` from base + paid payments.
- `savingsReleases[]` was audit-only — not in recompute.
- Buffer with `baseSaved = 0` and paid £60 contribution: release £32 clamped base to 0; recompute restored £60.

---

## Changes

### 1. `geodeSavingsReleaseDeductionSumForSource(state, sourceId)`

Sums release amounts where:

- `sourceId` matches
- `confirmedByUser !== false`
- `balanceMutationMode === 'event_derived'`

Legacy records without mode or with `base_delta` are **excluded** (deduction assumed already in base fields).

### 2. `geodeRecomputeBalancesFromPayments()`

After base + paid contributions:

- Goals: subtract event-derived release sum per `goal.id`
- Investments: subtract event-derived release sum per `investment.id`
- Clamp to ≥ 0

### 3. `geodeApplySavingsRelease()`

- **Removed** direct `baseSaved` / `baseBalance` mutation
- Appends release with `balanceMutationMode: 'event_derived'`
- Recompute → update `remainingBalance` on record

### 4. `geodeNormalizeSavingsReleases()`

- Preserves `balanceMutationMode` when present
- Defaults missing to `'base_delta'` (legacy compatibility)

---

## Legacy / double-deduction mitigation

| Record type | Recompute subtracts? | Why |
|-------------|------------------------|-----|
| New (`event_derived`) | Yes | Canonical model |
| Old (no mode → `base_delta`) | No | Base was mutated at apply time under 81D-3 |

**Local QA note:** Buffer test releases created before D.2.2 may be `base_delta` and will not deduct. Clear `savingsReleases[]` or re-release after upgrade.

Investment releases that already reduced `baseBalance` under old apply remain correct via `base_delta` default.
