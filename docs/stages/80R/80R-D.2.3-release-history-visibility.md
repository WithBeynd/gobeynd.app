## Objective

Add calm, factual, read-only visibility for Savings Release events so users can trace reserve-use continuity over time ‚Äî without shame, recovery framing, behavioural scoring, or affordability mutation.

---

## Pre-build audit (Task 1)

| Representation | Visible before | Notes |
|----------------|----------------|-------|
| `savingsReleases[]` | No UI | Canonical event ledger since 81D-3 / 80R-D.2.2 |
| `activityLog('savings_release')` | Write-only | Duplicate shadow; no reader |
| JSON backup | Included | `geodeBeyndBackupAllowedDataKeys()` |
| CSV export | Omitted | Uses materialized `goal.saved` only |
| `beyondStatement` | No release rows | Unrelated archive surface |
| `_geodeLastSavingsReleaseAudit` | Dev console only | Post-apply audit window |

**Stored metadata per release:** `id`, `sourceType`, `sourceId`, `amount`, `reason`, `date`, `ym`, `relatedYm`, `remainingBalance`, `createdAt`, `confirmedByUser`, `note`, optional `reserveClass` / `reserveTier` / `copyKey`, `balanceMutationMode`.

**Stale / duplicate:** `activityLog` duplicates amount + metadata but is trimmed (~75 days). History UI reads **`savingsReleases[]` only** (canonical).

## Implementation

| File | Change |
|------|--------|
| `index.html` | `geodeSavingsReleaseHistoryTypeLabel`, `geodeFormatSavingsReleaseHistoryEntry`, `geodeSavingsReleaseHistoryEntries`, `geodeSavingsReleaseHistorySectionHtml` |
| `index.html` | Settings ‚Üí **Data, privacy & backup** ‚Äî archival section when releases exist |

### Formatter rules

**Shown:** source name, factual type label (Buffer / Investment reserve / Saved amount), amount, date, balance after (if stored), footnote ‚ÄúRecorded as reserve use‚Äù, optional user note.

**Hidden:** `reason` codes, `reserveTier`, `reserveClass`, copy keys, emotional or recovery language.

### Surface placement

- **Settings ‚Üí Data, privacy & backup** ‚Äî between privacy card and Export & backup
- **Not:** Home, Plan, Reflection, Reality, Tier-C, Goals list, notifications

When `savingsReleases[]` is empty, **no section renders** (silence-first).
