## Objective

Define a **cross-surface pacing suppression contract** so Beynd does not narrate the same pacing, quiet, recovery, or softening meaning across Plan and Reflection (and adjacent surfaces) in the same user journey.

This stage defines **suppression law only**. It does **not** implement helpers, UI changes, or runtime wiring.

**Core principle:** Interpretation may exist without earning authority — and must not **stack** the same meaning across surfaces.

### Known gap (80N-N finding)

**Within-surface** collision metadata exists in `geodeStabilityRoomShadowMetadata` (`narratorCollisions`, `deduplicationReadiness`) but **does not read Plan tab state**. A user who sees a Plan pacing note and then opens Reflection may still see `month_so_far` gentle/recovering tone — **semantic duplicate, not a regression**.

Shadow metadata already documents intent:

```text
gentlePacing: 'plan_pacing_note_then_stability_if_absent_on_surface'
plan_pacing_note_owns_gentle_pace_on_plan_tab
```

**80N-N.1 makes this cross-surface law explicit and implementation-ready.**

## 2. Cross-surface ownership law

### Primary / secondary / forbidden / shadow-only

| Meaning class | Primary owner | Secondary owner (may echo, not re-interpret) | Forbidden owners | Shadow-only |
|---------------|---------------|---------------------------------------------|------------------|-------------|
| **PACING (gentle / smaller steps)** | **Plan pacing note** (when Plan visited or note would render) | Reflection `month_so_far` **only if Plan note absent on journey** | Home prose, Ask card, Tier-C pacing domain | `noticed.pacing` |
| **RECOVERY (one clear step)** | **Plan pacing note** (recovery line) | None live on Reflection | Reflection recovery paraphrase, second “one step” block | `noticed.recovery` |
| **QUIET / HELD_BACK disclosure** | **Reflection `background`** (opt-in explain) | Home muted lines (orientation, not full explain) | Plan tab, Tier-C when background active | `noticed.resilience` export |
| **MONTH_SO_FAR (observational tone)** | **Reflection atmosphere → month_so_far** | None on Plan | Plan pacing note duplicating month tone | — |
| **USER month intent** | **Reality capture → Reflection echo** | Carry suggestion (pre-fill only) | inferred flexibility copy, Ask diagnosis | `noticed.flexibility` |
| **BREATHING_ROOM softening explain** | **Plan detail footnote** (future `planPath`) **OR** Stability — **exclusive** | Step `memoryReasonCodes` (factual codes, not prose stack) | Home, Reflection atmosphere, Tier-C | `noticed.breathingRoomIntent` |
| **USER feeling posture** | **Reality feeling chips + moment body** | None inferring on other surfaces | Plan emotional narration, Reflection diagnosis | Posture in snapshot |
| **REVERSIBILITY** | **Plan banner / shift UX** | Ask hint (internal) | Shame framing anywhere | — |
| **Financial step truth** | **Plan (Tier 0)** | Plan detail transparency | Reflection amounts, Ask overrides | — |
| **Pressure / bands / scores** | **None (internal)** | — | **All UI surfaces permanently** | `noticed.pressure` |

### Surface responsibility (refined)

| Surface | Owns | Must NOT own |
|---------|------|--------------|
| **Plan** | Immediate step pacing, recovery line, step amounts, memory reason codes | Month-so-far tone, heldBack full disclosure, emotional diagnosis |
| **Plan detail** | Step-specific explanation, future breathing-room footnote | Cross-surface month narrative |
| **Home** | Orientation, one primary signal, muted defer **labels** | Deep pacing interpretation, month-so-far prose |
| **Reality** | User-authored context, session feeling capture | Pacing classification display, orchestration narrative |
| **Reflection** | Optional post-hoc interpretation, atmosphere blocks (max 3) | Plan math, predictive pacing, stacking Plan pacing meaning |
| **Stability Room** (future) | Calm explainability when Reflection/Plan absent on surface | Override Plan or Home policy |
| **Ask Beynd** | Conversational explain on request | Proactive stacking of all narrators |
| **noticed.*** | Observational vocabulary | Live copy without gate |

## 3. Suppression precedence (cross-surface)

When two surfaces could express the same meaning in one **user journey**, apply rules in order — **first matching rule wins**:

| Order | Rule | Rationale |
|-------|------|-----------|
| **1** | **User-authored Reality context** beats inferred interpretation | Agency (Continuity Law 3) |
| **2** | **Tier 0 financial truth** is never suppressed for interpretive convenience | Never hide amounts, overdue, payment status |
| **3** | **Silence** when meaning already expressed on **same surface** | Existing within-Reflection gate law |
| **4** | **Plan step-specific pacing** owns PACING/RECOVERY on Plan tab | Plan is execution surface |
| **5** | **Plan pacing note active on journey** → suppress Reflection PACING paraphrase in `month_so_far` | **80N-N.1 core rule** |
| **6** | **Atmosphere supremacy** inside Reflection | recovering/reflective suppress Tier-C |
| **7** | **Background block** beats Tier-C resilience | Existing Tier-C law |
| **8** | **Reflection background** owns QUIET disclosure when user opts in → do not repeat full heldBack story on Home in same session | Explain once, calmly |
| **9** | **Home muted lines** stay minimal when Reflection entry available | Orientation not narration |
| **10** | **Ask Beynd** deferential to surfaces user already visited | No “as I said on Plan and Reflection…” stacking |
| **11** | **Shadow exports** never override live copy | Tier 2 non-authoritative |

### Journey definition (for suppression)

**User journey** = from app open or tab switch until:
- session end, OR
- 30-minute idle (recommended default for implementation), OR
- explicit dismiss of Reflection entry

**Journey memory for suppression:** **session-scoped only** — `sessionStorage` flags or in-memory compose cache. **No** cross-day profiling.

## 4. “Same idea” detection contract

### Semantic buckets (cross-surface)

| Bucket ID | Trigger phrases / signals | Primary owner |
|-----------|---------------------------|---------------|
| `IDEA_GENTLE_PACE` | “smaller steps”, “gentle pace”, “sequence stays the same”, gentle `pacingMode` | Plan pacing note |
| `IDEA_ONE_CLEAR_STEP` | “one clear next step”, recovery emerging | Plan pacing note |
| `IDEA_QUIET_HOME` | “kept quiet”, “background”, “stayed in the background”, heldBack | Reflection background OR Tier-C (not both) |
| `IDEA_MONTH_STEADY` | “reads steady”, “fits together”, calm atmosphere | Reflection month_so_far |
| `IDEA_TIGHT_MONTH` | “tighter”, “narrow band”, negative leftover tone | Plan note OR Home alert — not Reflection mood stack |
| `IDEA_SLOW_DOWN` | feeling slow/unsure moment body | Reality feeling — not Plan + Reflection duplicate |
| `IDEA_MEMORY_SOFTEN` | memory reason codes / future footnote | Plan detail exclusive with Stability |
| `IDEA_USER_PROTECT` | protect/worried/success strings | Reality echo only |

### Classification matrix

| Relationship | Example | Verdict | Action |
|--------------|---------|---------|--------|
| **Exact duplicate** | Plan “smaller steps…” + Reflection month_so_far gentle tone same session | **Semantic duplicate** | SUPPRESS Reflection PACING paraphrase |
| **Semantic duplicate** | Plan “one clear step” + Tier-C quiet line same session | **Semantic duplicate** | Tier-C already suppressed if background; else SUPPRESS Tier-C |
| **Acceptable reinforcement** | User protect echoed in `what_matters` only | **Safe echo** | USER_ECHO_ONLY |
| **Safe echo** | Plan “protecting [goal]” + Reflection plan_protecting orientation | **Acceptable** | ALLOW (different class: orientation vs pacing) |
| **Forbidden stacking** | Home muted + Reflection background + Tier-C all QUIET | **Emotional stacking** | SUPPRESS Tier-C; cap background |
| **Forbidden** | “You usually go slow in month two” | **Profiling** | SHADOW_ONLY / never ship |

### Detection inputs (read-only, no new inference)

Future helper may use **only**:

- Boolean: `planPacingNoteRenderedThisJourney`
- Boolean: `planTabVisitedThisJourney`
- Enum: `lastPacingBucketExpressed` (session)
- Reflection `atmosphere`, `blockMap` ids
- `heldBackSignals`, `showBackground`
- User month context presence
- **Not allowed:** `often*`, streak, usual-month, pressure dimensions

## 5. Suppression states

| State | When used | Surface applies | Visible copy change? | Runtime state change? | Persistence |
|-------|-----------|-----------------|------------------------|-------------------------|-------------|
| **ALLOW** | No duplicate; owner surface | Any | Shows copy | None | None |
| **SUPPRESS** | Duplicate detected; silence preferred | Reflection, Ask, Tier-C | Hides candidate copy | None | Session journey flag only |
| **DEFER** | Meaning deferred to another surface | Home muted → Reflection | Shorter Home copy | None | None |
| **REPLACE** | Tier-C replaces month_so_far (existing law) | Reflection only | Swaps blocks | None | None |
| **USER_ECHO_ONLY** | User words; no inference | Reflection `what_matters` | Echo verbatim | None | None |
| **SHADOW_ONLY** | noticed.*, pressure, often* | Export/debug | Never user-visible | None | Debug window refs only |

**Hard rule:** `persistenceAllowed: false` for all suppression decisions except session journey flags (ephemeral).

## 6. Future implementation audit contract (spec only)

### Proposed helper signature (DO NOT BUILD in 80N-N.1)

```js
geodeCrossSurfacePacingSuppress(snapshot, surface, candidate, opts)
```

| Parameter | Type | Notes |
|-----------|------|-------|
| `snapshot` | Stability snapshot or null | Read-only |
| `surface` | `'plan' \| 'reflection' \| 'home' \| 'reality' \| 'ask'` | Caller surface |
| `candidate` | `{ meaningBucket, blockId?, copyPreview?, tier? }` | Candidate narration |
| `opts` | `{ journeyContext, planPacingNoteActive?, blockMap? }` | Session-scoped |

### Return contract

```js
{
  version: 1,
  stage: '80N-N.1+',
  verdict: 'ALLOW' | 'SUPPRESS' | 'DEFER' | 'REPLACE' | 'USER_ECHO_ONLY' | 'SHADOW_ONLY',
  reason: string,                    // machine code e.g. 'plan_pacing_note_owns_journey'
  canonicalOwner: string,            // e.g. 'plan_pacing_note'
  duplicateOf: string | null,        // bucket id or surface.block
  recommendedAction: string,         // human-readable for debug
  renderAllowed: boolean,
  copyAllowed: boolean,
  persistenceAllowed: false,         // always false
  authorityTier: 0 | 1 | 2 | 3
}
```

### Implementation constraints

- **Read-only** — must not mutate `S`, Plan, orchestration, or atmosphere
- **No afford / getMonthPlan calls** from suppress helper
- **Default verdict:** SUPPRESS when uncertain (silence-first)
- **Debug export:** `window._geodeCrossSurfacePacingSuppressLast` optional

## 7. Forbidden behaviours

Explicitly **forbidden** in any future implementation:

| Forbidden | Why |
|-----------|-----|
| Suppressing financially important information (overdue, negative leftover facts, payment status) | Tier 0 truth |
| Reflection overriding Plan amounts or sequence | Authority inversion |
| Plan narrating deep emotional state | Plan owns execution |
| Home repeating Reflection month-so-far meaning | Parallel narrators |
| Ask Beynd stacking Plan + Reflection + Home context unprompted | Surveillance feel |
| “Usual month” / journey profiling across days | Continuity Law 11 |
| Pressure bands or dimension levels in UI | INTERNAL_ONLY permanent |
| Persistent pacing profile in `S` or `localStorage` | Profiling |
| Emotional scoring or hesitation inference | Anti-profiling |
| Using suppression to hide memory softening from Plan detail | Explainability Law 5 |
| Cross-surface suppress without session journey scope | Hidden long-memory control |
