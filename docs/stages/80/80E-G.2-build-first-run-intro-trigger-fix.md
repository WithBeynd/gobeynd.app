**Type:** Implementation

## Objective

Make `pg-fr` first-run intro reliably show after onboarding when the user has not seen it before.

**Rule:** Onboarding sets up the user. First-run intro orients the user.

---

## Scope

| File | Change |
|------|--------|
| `index.html` | `maybeShowFirstRunIntro` gate; `wipeLocalAppStateAndReload` clears `FR_KEY` |

---

## Implementation

### `maybeShowFirstRunIntro`

**Before:** required `_geodeFirstBoot` (page-load absence of `geode_v6`) **and** `!hasSeenFirstRunIntro()`.

**After:** requires `S.hasOnboarded === true` **and** `!hasSeenFirstRunIntro()` only.

`_geodeFirstBoot` left in `load()` unchanged (unused by intro gate after this fix).

### Reset All Data

`wipeLocalAppStateAndReload()` now also `localStorage.removeItem('geode_fr_intro_v1')` so Reset All Data restores full first-run path including intro.

`FR_KEY` name unchanged: `geode_fr_intro_v1`.
