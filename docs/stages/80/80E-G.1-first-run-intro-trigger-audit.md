# Stage 80E-G.1 — First-Run Intro Trigger Audit

**Type:** Read-only flow audit  
**Verdict:** NEEDS ATTENTION  
**Date:** May 2026  
**Parent:** [80E-G build](./80E-G-build-first-run-intro-copy.md)

---

## Objective

Verify whether `pg-fr` first-run intro is reachable after onboarding and whether new users actually see it before landing on Home.

---

## Core finding

80E-G copy in `#fr2` / `#fr3` is present and correct, but **`maybeShowFirstRunIntro()` is gated by `_geodeFirstBoot`**, set once at page load as `!localStorage.getItem('geode_v6')`. That flag diverges from the onboarding path:

| Scenario | Intro shown? |
|----------|--------------|
| Mid-onboarding page refresh (after `ob1Next` save) | **No** — KEY exists at reload → `_geodeFirstBoot = false` |
| Reset All Data with stale `geode_fr_intro_v1` | **No** — `FR_KEY` not cleared on wipe |
| Upgraded user, KEY present, never saw intro | **No** — `_geodeFirstBoot = false` even if `FR_KEY` unset |
| True fresh install, one session, no reload | **Yes** — narrow path only |

Observed user flow **ob1 → ob2 → ob3 → Home** (skipping fr1–fr3) matches code when either gate fails.

---

## Trigger implementation (pre-fix)

```js
function maybeShowFirstRunIntro() {
  if (!_geodeFirstBoot) return;      // GATE 1 — page-load KEY absence
  if (hasSeenFirstRunIntro()) return; // GATE 2 — FR_KEY === '1'
  ...
}
```

Called from `launchApp()` after `goTab('home')` — paths: `ob3Finish`, `hideSplash` (returning user), `doUnlock`.

---

## Recommendation

1. **Decouple intro from `_geodeFirstBoot`** — show when `S.hasOnboarded === true` and `!hasSeenFirstRunIntro()`.
2. **Clear `FR_KEY` in `wipeLocalAppStateAndReload()`** so Reset All Data matches first-run expectations.
3. Preserve `FR_KEY = 'geode_fr_intro_v1'` and fr1–fr3 copy unchanged.
