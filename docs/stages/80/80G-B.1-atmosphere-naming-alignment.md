# Stage 80G-B.1 — Atmosphere Naming Alignment

**Type:** Product-language hardening  
**Verdict:** Built (local)  
**Date:** May 2026  
**Parent:** [80G-C post-build audit](./80G-B-build-stability-room-visual-model.md) — atmosphere naming NEEDS ATTENTION

---

## Objective

Rename Stability Room visual model atmosphere token **`opportunity` → `room`** so Reflection layer language does not imply spare money, spendable capacity, or nudge-style action opportunity.

**Not `breathing_room`** — that token already means reserved affordability floor in plan logic.

---

## Change

| Before | After |
|--------|-------|
| `atmosphere: 'opportunity'` when `growth_room` posture | `atmosphere: 'room'` |

**Unchanged:** VI registers, section visibility rules, entry copy, snapshot helper, UI, routes, CSS.

---

## Scope

| File | Change |
|------|--------|
| `index.html` | Three string comparisons in `geodeStabilityRoomVisualModel` only |
| Docs | Atmosphere token list + stage map |

---

## Rationale

- **80A / 80G-C:** Stability Room must not imply funds available.
- **`growth_room` posture** already labels as “growth room” in Ask Beynd context — atmosphere **`room`** aligns.
- Home nudge system uses **`opportunity`** as topic/type — separate domain; Reflection atmosphere must not collide.
