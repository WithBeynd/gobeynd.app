# 91D-B.1 — Plan Intro Rationale Truth Alignment Fix

**Verdict:** PASS  
**File:** `index.html` — `rPlan()` Plan intro rationale block (~35464–35486)

## Problem

Plan intro falsely claimed "No high-interest debt flagged" when 12% APR debt existed but Monthly Left was only £30. Root cause: `_hD` combined debt existence with `calcMonthlyLeftover(S) > 50`, so low-left months fell through to a buffer branch that denied debt and claimed "you have room."

## Change (copy/gate only)

- Split `_hasHighAprDebt` from `_hasRoomForDebtStep` (`_lft > 50`).
- Replaced single false `_bSh` fallback with branches A–G (debt-first, implicit vs explicit buffer, tight-month gating).
- Removed goals branch that could say "more room for goals" at low Monthly Left.
- Default G when no branch matches.

**Protected (unchanged):** `getMonthPlan`, `calcMonthlyLeftover`, Stage 62, debt sequencing, buffer targets, Reflection, orchestration producers.
