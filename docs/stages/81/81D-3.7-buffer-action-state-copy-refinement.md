# Stage 81D-3.7 — Buffer Action State Copy Refinement

**Type:** Fix (copy only)  
**Verdict:** Built — audit pending  
**Date:** May 2026  
**Parent:** [81D-3.6 buffer adequacy Plan gate fix](./81D-3.6-buffer-adequacy-plan-gate-fix.md)  
**Closes:** Buffer action copy did not distinguish add-buffer vs contribute-to-buffer after 81D-3.5 asset truth

---

## Objective

Refine buffer **action labels and headlines** so Beynd says what the user is actually doing:

1. **Add** emergency buffer (no goal yet)
2. **Contribute** to an existing emergency buffer (goal exists, not adequate)

Copy-only. No Plan amounts, sequencing, payment save, balances, or Savings Release changes.

---

## New helpers (display only)

| Helper | Role |
|--------|------|
| **`geodeBufferHasScheduledUnpaidBuffer(state)`** | Detects unpaid buffer contribution this month |
| **`geodeBufferActionCopy(state)`** | Returns headline, CTA, card, plan, modal copy from goal + readiness |
| **`geodeIsBufferMoneyAffordCta(cta)`** | Afford gate recognises new buffer CTAs |

---

## Copy matrix

| State | Headline | CTA | Modal heading |
|-------|----------|-----|---------------|
| No emergency goal | Add your emergency buffer | Add emergency buffer | Add emergency buffer |
| Goal exists, not scheduled | Build your emergency buffer | Contribute to buffer | Contribute to buffer |
| Goal exists + scheduled unpaid | Your buffer is still building | Add to buffer | Add to buffer |
| Estimated buffer, no goal yet | Build / still building (by schedule) | Add emergency buffer / Add to buffer | Same family |
| Buffer complete | Hidden (existing gate) | — | — |

Payment **name** prefill stays **`Emergency buffer`** (81D-3.5 create/link on save unchanged).
