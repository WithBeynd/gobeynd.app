# Stage 81D-3.5 — Auto-create / Link Emergency Buffer Goal

**Type:** Build (truth fix)  
**Verdict:** Built — audit pending  
**Date:** May 2026  
**Parent:** Stage 81D-3.4 buffer contribution truth audit (read-only FAIL)  
**Closes:** Buffer flow/balance gap — buffer contributions without `goalId` did not increase saved assets, net worth, or Savings Release eligibility

---

## Objective

When Beynd suggests or schedules a buffer contribution, the saved payment must **create or link** a real emergency buffer **goal** so that **completed** contributions increase `goal.saved`, net worth, and (later) Savings Release eligibility.

**Core principle:** If Beynd says the user is building a buffer, Beynd must create or link a real buffer asset.

**Forward-only:** No backfill of existing payments; editing an old unlinked payment does not auto-link.

---

## Changes

| Area | Change |
|------|--------|
| **`geodeEnsureEmergencyBufferGoalForPayment(state, opts)`** | Detects buffer payment (`bufferContribution` flag or `geodePaymentNameLooksLikeBuffer`); skips if already linked or linked-investment emergency goal exists; reuses `getFirstEmergencyGoal` or creates `Emergency buffer` goal (`cat: emergency`, target `3 ×` monthly expenses when known) |
| **`geodeSavePayApply`** | Before persist, when new payment or buffer CTA session and no goal/invest/debt link, calls helper; sets `goalId` + `payKind: 'goal'` when applied |
| **`openPayModal`** | Session flag `window._geodePayPrefillBufferContribution` from prefill; cleared on edit-existing and after apply |
| **Buffer CTAs** | `openSuggestedAction` + `geodeMainActionFromPriorityStep` prefill `bufferContribution: true` on `openPayModal` |

---

## Helper behaviour

**Applies when:**

- `opts.bufferContribution === true` or `opts.fromBufferContribution === true`, **or**
- Payment name matches `geodePaymentNameLooksLikeBuffer`

**Skips when:**

- `goalId`, `investId`, or `debtId` already set
- Existing emergency goal has positive linked investment (`geodeGoalHasPositiveLinkedInvestmentForState`) — user must link manually

**Creates goal when none exists:**

- `name: 'Emergency buffer'`, `cat: 'emergency'`, `saved` / `baseSaved: 0`
- `amount: 3 × geodeMonthlyExpenseTotal` when expenses exist, else `0`
- `isPrimary: true` when first goal in list

**Returns:** `{ applied, goalId, created, linkedExisting, reasonCodes[] }`

---

## Hook points

1. **Home Main Action** — `geodeMainActionFromPriorityStep` buffer branch → `openPayModal(null, { …, bufferContribution: true })`
2. **Suggested actions** — `openSuggestedAction` `buffer_contribution` prefill
3. **Save path** — `geodeSavePayApply` (new payments only, or explicit buffer CTA session)

Paid goal-linked contributions continue through existing **`geodeRecomputeBalancesFromPayments`** (unchanged).

## Manual QA

1. **No emergency goal** — Plan buffer contribution from Home → save → goal appears; mark paid → goal saved + net worth up; Savings Release shows buffer source when eligible.
2. **Existing emergency goal** — Buffer CTA → payment gets existing `goalId`; no duplicate goal.
3. **Linked-investment emergency goal** — Buffer CTA → payment stays unlinked bill (no auto-link); user can pick link manually.
4. **Edit old unlinked “Emergency buffer” payment** — Save without changing link → still unlinked (no retroactive link).
5. **Manual new payment** named “Emergency buffer” → auto-links or creates goal on save.
