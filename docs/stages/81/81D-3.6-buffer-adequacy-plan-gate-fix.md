**Type:** Fix (Plan regression)  
**Verdict:** Built — audit pending

---

## Objective

Fix the Plan regression from **81D-3.5**: scheduling a buffer contribution creates a real Emergency Buffer goal, but `getMonthPlan` previously hid the buffer step as soon as **any** emergency goal existed.

**Core principle:** If Beynd says the user is building a buffer, the Plan must continue until the buffer is meaningfully funded or adequate. Goal existence alone is not enough.

---

## Changes

| Area | Change |
|------|--------|
| **`getMonthPlan` Step 2 gate** | Replaced `!hasEmergency` with `geodeGetBufferReadinessState(S) !== 'BUFFER_COMPLETE'` |
| **`remainingEmergencyNeeded`** | Uses `getFirstEmergencyGoal` + `geodeGoalEffectiveSavedFromState`; target falls back to `3 × geodeMonthlyExpenseTotal` when `goal.amount <= 0`; subtracts scheduled unpaid buffer contributions this month |

---

## Gate logic (after)

**Show buffer step when:**

- Readiness is not `BUFFER_COMPLETE`, **and**
- `budget > 0`, **and**
- `steps.length < 4`, **and**
- Suggested amount `ef > 0` after cap

**Hide buffer step when:**

- `saved >= target` (readiness `BUFFER_COMPLETE`), **or**
- `target − saved − scheduledUnpaid <= 0` (adequate including scheduled intent), **or**
- No plan room after debt step

**Unchanged:** debt step order, goal catch-up, invest step, budget ratios, `geodePlanStepProgress` display helpers.
