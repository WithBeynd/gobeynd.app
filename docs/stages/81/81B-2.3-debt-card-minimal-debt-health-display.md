## Implementation

### Removed from debt card

All narrative / guidance copy, including:

- `geodeDebtProviderReviewNudgeCopy` on card (interest/fees paragraphs, “worth a quick look”, etc.)
- Long status line (`Save your provider balance here when you check in`, `Provider balance saved …` combined with APR type)
- Multi-line layout

### Added — `geodeDebtHealthCardMinimalCopy(debt, state)`

Single factual line when `geodeDebtShouldShowHealthOnCard` is true (81B-2.2 gate unchanged):

| Condition (priority) | Card line |
|----------------------|-----------|
| Paid payment since last save | `Payment since last balance confirmation` |
| Never confirmed | `Balance not confirmed` |
| Stale confirmation | `Balance confirmed {n}d ago` |
| Variable APR (fresh save) | `Variable APR` |
| Unknown APR type (fresh save) | `APR type not set` |

No buttons, no second line, no coaching paragraphs.

### `geodeDebtHealthCardHtml`

Renders at most **one** muted `9px` line via minimal copy helper.

## Behaviour unchanged

| Surface | Unchanged |
|---------|-----------|
| Edit Debt modal | Inline provider/balance guidance, APR type selector, **`geodeDebtProviderReviewStatusCopy`** when reviewed (not **`geodeDebtProviderReviewNudgeCopy`**) |

| Principal estimates | Shadow helpers only; **no card UI** |

## Validation

```js
var html = geodeDebtHealthCardHtml({ id: 'x', interestType: 'unknown', balance: 1000 }, S);
html.indexOf('Worth a quick look') < 0;
html.indexOf('Save your provider balance') < 0;
html.indexOf('Your provider may add interest') < 0;
html.indexOf('Balance not confirmed') >= 0;

typeof geodeDebtHealthCardMinimalCopy === 'function';
```

Manual QA:

1. New debt → one short line (`Balance not confirmed` or `APR type not set`)
2. Stale save → `Balance confirmed Nd ago` only (no nudge paragraph)
3. Edit Debt → inline provider copy + APR selector + status line when reviewed
4. Fixed + fresh save + no post-payment trigger → no card line
