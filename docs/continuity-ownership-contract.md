## 1. What continuity means in Beynd

### Philosophical definition

Beynd continuity is **adaptive emotional-financial support across time** — not a record of user performance.

The month **evolves**. The user does not **fail** the plan. When circumstances shift, Beynd holds orientation, preserves reversibility, and adapts without punishment, surveillance, or hidden control.

Continuity means:

- **Adaptive support** — amounts and pacing may soften when pressure is present; sequence and truth remain.
- **Evolving month context** — what matters this month can be named, updated, and carried forward as suggestion.
- **Calm financial orientation** — debt, buffer, goals, and invest remain legible without urgency stacking.
- **Reversible guidance** — reality shift, month context, and shifted views stay optional and easy to reverse.
- **Non-punitive adaptation** — no failure framing, no behavioural grades, no “you dropped off” language.
- **Pacing-aware interpretation** — gentle pace is named when helpful; it never becomes invisible throttling.
- **Continuity of emotional safety** — silence, atmosphere, and deferral protect load before explanation.
- **Continuity of financial orientation** — Plan truth and payment truth remain stable anchors.

Continuity is **not**:

- Behavioural profiling or personality modelling
- Predictive psychology or “usual-you” inference
- Invisible optimisation or hidden reserve systems
- Emotional dependency loops or therapy framing
- Hidden behavioural steering or orchestration mutation
- Surveillance-feeling memory or “AI knows you” systems
- A budgeting tracker mindset (scorekeeping, streak shame, productivity framing)

### Behavioural definition

At runtime, continuity is expressed through **bounded layers**:

| Layer | Continuity expression |
|-------|----------------------|
| **Capture** | User-authored month context; optional feeling chips; balance sense-check |
| **Persistence** | Month-scoped context; optional carry suggestion; confidence-gated financial memory |
| **Interpretation** | Atmosphere, pacing note, Reflection blocks — observational, capped, suppressible |
| **Adaptation** | Plan softening via disclosed memory modifiers; reality shift preview — reversible |
| **Silence** | Home deferral, quiet periods, Tier-C suppression — continuity through restraint |

Continuity **must never** become a single inferred “user state” object that drives multiple surfaces without user-authored input or Tier-0 math.

### Runtime implications

1. **One canonical owner per signal** — see ownership matrix below.
2. **User words dominate** — protect/worried/success echo before inferred posture.
3. **Interpretation is non-authoritative** — `noticed.*` is Tier 2; atmosphere is display register, not policy.
4. **Behavioural mutation requires Tier 0 contract** — only Plan math, orchestration visibility, and explicit user actions may change execution truth.
5. **Cross-surface narration requires collision law** — parallel narrators are a governance failure unless suppressed.

### Forbidden drift patterns

| Pattern | Why forbidden |
|---------|---------------|
| Multiple surfaces explaining the same pressure | Feels analysed; erodes trust |
| Inferred continuity replacing user context | Surveillance; removes agency |
| Memory softening without explainability path | Hidden control |
| Persistent emotional model in `S` | Profiling |
| “Usual month” or pattern prediction | Predictive psychology |
| Tier 2 → live UI without gate + copy map | Interpretation becomes authority |
| Orchestration changes without documented stage | Hidden behavioural drift |

## 2. Canonical ownership matrix

### Summary table

| System | Canonical responsibility | Tier |
|--------|-------------------------|------|
| **Plan** | Deterministic financial sequencing + execution truth | 0–1 |
| **Reality** | User-authored month awareness + sense-check capture | 0 capture / 1 display |
| **Home** | Orientation + visibility arbitration (not deep interpretation) | 1 |
| **Reflection** | Calm optional month understanding (opt-in) | 3 |
| **Stability Room** | Future explainability surface (not yet routed) | 3 (future) |
| **noticed.*** | Observational vocabulary export | 2 |
| **orchestration** | Visibility arbitration + deferral policy | 1 (policy) |
| **atmosphere** | Emotional register for interpretive surfaces | 1 (display) |
| **financial memory** | Continuity softening only (confidence-gated) | 0 effect / 1–2 explain |

---

### Plan

| Dimension | Rule |
|-----------|------|
| **Owns** | Step sequence, amounts, payment truth, action-state display, `bufferPreference` sizing, pacing note paraphrase (from PAO) |
| **Must never own** | Emotional diagnosis, quiet policy, month meaning, user personality, `noticed.*` consumption, suppression explanation |
| **May observe** | Affordability, posture inputs, memory modifiers (via existing contracts only) |
| **May narrate** | Yes — pacing note, plan detail, alignment summary (Tier 1, deterministic paraphrase) |
| **May persist** | No interpretation persistence; Plan state is financial truth only |
| **May mutate behaviour** | Yes — Tier 0 math and sizing only, via established helpers |

### Reality

| Dimension | Rule |
|-----------|------|
| **Owns** | `monthContext` (protect/worried/success), balance sense-check, feeling chip capture (session), carry affordance, shift preview UX |

| **May observe** | Plan preview for shift alignment; carry from prior month |

| **May persist** | Month context in `S`; carry in `S`; feeling in session only (see memory-boundaries) |

## 3. Authority hierarchy

When signals conflict, resolve in this order:

1. **Tier 0 truth** — Plan math, payments, afford baseline  
2. **User-authored context** — month context, explicit shift approval  
3. **Orchestration silence** — deferral before explanation  
4. **Atmosphere register** — emotional tone for display  
5. **Tier 1 paraphrase** — pacing note, Home muted lines  
6. **Tier 2 observation** — `noticed.*` (non-authoritative)  
7. **Tier 3 opt-in** — Reflection / Stability (suppression-first)

**Interpretation must never quietly become authority.** Any promotion from Tier 2 → Tier 3 → Tier 1 → Tier 0 requires a documented governance stage.
